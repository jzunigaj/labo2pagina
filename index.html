Resumen

En esta parte del proyecto, se seleccionaron dos proyectos de software (uno en C++ y otro en Python) con el objetivo de generar documentación automática utilizando herramientas especializadas: Doxygen para C++ y Sphinx para Python. La documentación generada fue publicada utilizando GitHub Pages y Read the Docs respectivamente. Donde ambos programas fueron generados por chatgpt por conveniencia con el tiempo.

Proyecto en C++ – GestorDeTareas

Descripción

Se desarrolló un sistema sencillo de gestión de tareas en consola. El usuario puede agregar tareas, marcarlas como completadas, listarlas y eliminarlas. Todo el código fue debidamente comentado utilizando el estilo compatible con Doxygen.

Estructura del proyecto

GestorDeTareas/
├── main.cpp
├── Tarea.h
├── Tarea.cpp
└── Doxyfile

Ejemplo de comentario Doxygen:


/**
 * @brief Clase que representa una tarea individual.
 */
class Tarea {
public:
    /**
     * @brief Constructor de la clase Tarea.
     * @param descripcion Descripción de la tarea.
     */
    Tarea(std::string descripcion);

    /**
     * @brief Marca la tarea como completada.
     */
    void completar();

    /// Imprime los detalles de la tarea.
    void mostrar() const;
};

Proceso para generar la documentación con Doxygen

  1. Instalar Doxygen y Graphviz:
    sudo apt install doxygen graphviz
  2. Generar archivo de configuración:
    doxygen -g Doxyfile
  3. Editar Doxyfile para activar HTML:
    • EXTRACT_ALL = YES
    • GENERATE_HTML = YES
    • INPUT = .
    • RECURSIVE = YES
  4. Ejecutar Doxygen:
    doxygen Doxyfile

Proyecto en Python – CalculadoraAvanzada

Descripción

Se desarrolló una calculadora modular en Python que incluye funciones básicas (suma, resta, multiplicación, división) y avanzadas (potencia, raíz cuadrada, factorial). Todo el código fue documentado con docstrings en formato reStructuredText, compatible con Sphinx.

Estructura del proyecto

CalculadoraAvanzada/
├── src/
│   └── calculadora.py
├── docs/
│   ├── conf.py
│   ├── index.rst
│   └── _build/html/
└── README.md

Ejemplo de docstring para Sphinx


def sumar(a, b):
    '''
    Suma dos números
    :param a: Primer número.
    :param b: Segundo número.
    :return: Resultado de la suma.
    '''
    return a + b

Proceso para generar la documentación con Sphinx

  1. Instalar Sphinx:
    pip install sphinx
  2. Crear documentación:
    sphinx-quickstart
  3. Agregar la ruta del código en conf.py: 
    import os
import sys
sys.path.insert(0, os.path.abspath('../src'))
  4. Generar archivos .rst:
    sphinx-apidoc -o source/ ../src/
  5. Compilar la documentación:
    make html

Comentario sobre la experiencia

La generación de documentación automática fue una experiencia valiosa para entender cómo se estructuran proyectos profesionales.

Doxygen resultó muy fácil de usar. Su sistema basado en comentarios tipo /** */ es intuitivo. La generación de la documentación es rápida y muy visual. Lo más destacable fue la capacidad de integrar diagramas de clases gracias a Graphviz.

Sphinx, por otro lado, fue un poco más complejo al inicio, ya que requiere configurar varios archivos y conocer el formato reStructuredText. Sin embargo, ofrece una documentación extremadamente profesional y flexible. Read the Docs facilitó la publicación en línea de manera automática al enlazar el repositorio.

Ambas herramientas son muy recomendadas y demuestran la importancia de escribir comentarios adecuados en el código para facilitar la generación de documentación comprensible y profesional.
